# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

A layout simulator ("佈局模擬器") for the game 口袋學院物語2 (Pocket Academy 2), letting players plan school maps on a tile grid and see which "人氣景點" (popularity spots) their layout produces. The entire app is a single self-contained file, `index.html` — HTML, CSS, and vanilla JavaScript with no build step and no dependencies to install. **After any change, run `node scripts/check.js`** — it validates JS syntax and data consistency (icons/JP-names coverage, spot & dev-guide id integrity, built-in map shapes, share-code roundtrip); all checks must PASS. Project skills: `/pa2-check` (run the checker) and `/add-facility` (the standard workflow when the user reports a facility size, rename, or missing facility). Tailwind CSS and Google Fonts are loaded from CDNs, so the page needs internet access to render correctly.

To run: open `index.html` in a browser (e.g. `start index.html` from PowerShell). UI text is Traditional Chinese (zh-TW). Deployed to GitHub Pages (the `index.html` name is required for that); replace the `USERNAME.github.io/REPO` placeholders in `<head>` (canonical/og:url/og:image) with the real URL once known. Touch input is supported (tap paints, drag paints continuously, long-press is the eyedropper) and the layout stacks vertically under 768px. The map can also be shared via URL hash (`#m=` + RLE/base64url of the grid, see `encodeMap`/`decodeMap` — TYPE_KEYS order must stay append-only).

## Architecture (all inside 2.html's single `<script>` block)

- **`items`** — registry of every placeable tile type (id, display name, one-character `short` label, color, palette category `type`, optional `textCol`). Categories map 1:1 to the sidebar palette sections: `tool`, `env` (環境地形), `fac` (生活與設施), `spec` (教室與專科), `sports` (運動與社團), `farm` (動植物農牧). Adding a new building = adding one entry here; the palette renders itself from this object.
- **`SPOTS`** — the game's official 29 popularity spots (sourced from the カイロパーク攻略 Wiki). Each `req` entry is an item id or an array of ids meaning "any one of these" (e.g. 森林/櫻花樹). A spot forms when all three required facilities touch a common **4×4 window** (`checkSpots()`); the designated area is then each required facility's tiles in that window plus a 1-tile ring (2-tile ring for footprints ≥4 cells). `bonus` holds the attribute gains (文系/理系/體育/頭腦/運動/人氣/態度) and optional `teacher` the recruitable teacher(s), both shown in the hover tooltip.
- **`HANDBOOK`** — static in-game reference data (attack tips, school ranks, teachers, clubs, careers, events, special lessons, challenges, items) rendered as a tabbed modal via `buildHandbook(section)`, sharing the `reportModal` shell with `buildReport()` and `buildDevGuide()` (which lists facility unlock conditions per school-rank stage, auto-checking "×N prerequisite" counts against the map via `countBuilt()`). All data sourced from the カイロパーク攻略 Wiki. `JP_NAMES`/`SPOT_JP` map item/spot ids to Japanese names (shown in palette tooltips and the handbook's 中日對照 tab); the UI itself stays Traditional Chinese. The `<head>` carries SEO meta (description/keywords/Open Graph/JSON-LD WebApplication) — when hosting publicly, add `og:url`, `og:image`, and a canonical link once the domain is known. Marked tiles get a yellow ring and colored indicator dots.
- **`gridData`** — the single source of truth: a 26×24 array of `{ type, elevation }`. All rendering (`updateTileVisuals`, `updateSpotIndicators`) reads from it; `applyTool` writes to it and re-runs `checkSpots()` over the whole grid.
- **Coordinate system**: internal indices are `r` (0–25) / `c` (0–23), but the displayed axis labels replicate the game's coordinates — X = `r + 2` (2–27, top to bottom) and Y = `25 - c` (25 down to 2, left to right). Keep this mapping in mind when translating between on-screen labels and array indices.
- **Elevation**: each cell has `elevation` 1–3 (平地/高地1層/高地2層), chosen via `setElevation()` and applied when painting. Slopes are **derived, not stored**: `isSlope(r,c)` — an empty cell whose elevation is exactly one above a 4-neighbor (game rule: the whole boundary becomes slope, on the higher tile; a 2-level gap is a cliff). Slope cells get striped styling (`.slope-tile`), count as the `slope` item in spot checks (黃昏 spot), and the `slope` item is `hidden: true` — never placeable from the palette.
- **Persistence**: named saves go to `localStorage` under the key `pa2_maps_v9`; import/export copies the raw `JSON.stringify(gridData)` string via the clipboard. UI preferences (diamond view, selected tool, elevation, last save name) persist in `pa2_prefs_v1`; the canvas autosaves (debounced) to `pa2_autosave_v1` after every change and is restored on startup — the 健康鎮 preset only loads when no autosave exists. `PRESETS` lists the game's 5 towns (健康鎮/東部小鎮/湖畔鎮/溪谷鎮/百靈山丘); only 健康鎮 has map data so far — entries with `data: null` render as disabled "待匯入" options. Replace a `null` with an exported save-code string to enable that town. Selectable via the `presetSelect` dropdown, loaded with `loadPreset(key)`. The player's in-game progress lives in normal localStorage saves; `seedProgressSave()` injects a one-time "健康鎮進度" save from `PROGRESS_SEED_DATA` on first launch. On startup the 健康鎮 preset is auto-loaded.
- **Responsive grid**: tile size is driven by CSS vars `--tile`/`--axis`; `fitGrid()` recomputes them on load/resize so the whole map fits the viewport without scrollbars. A fixed `#coordHint` bar shows the hovered tile's game coordinates, type, and elevation (updated in the tile `mouseenter` handler and `applyTool`).

Painting works by mousedown + drag (`isDrawing` flag with `mouseenter` on tiles). Additional editing tools: rectangle fill (`rectMode` toggle or Shift+drag, 1×1 items only, commits on mouseup), right-click eyedropper (`contextmenu` → `selectTool`), and whole-building erase (empty tool on a cell with a `bid` removes every cell sharing that bid). `exportImage()` renders gridData to a canvas and downloads a PNG. `buildReport()` opens a modal summarizing active spots, total attribute bonuses (parsed from `bonus` strings), near-miss spots, and blocked-building count (`lastSpotStats`/`lastBlockedCount` are cached by `checkSpots`/`updateAccessWarnings`). Saves in `pa2_maps_v9` are stored as `{data, ts}` (legacy plain-array entries still load); the save dropdown shows timestamps and a rename button moves entries. Each stroke pushes a full-grid JSON snapshot onto `undoStack` (`pushHistory()`, Ctrl+Z / Ctrl+Y, max 60). Multi-tile buildings get a per-placement `bid` (block id) stored in their cells; `sameBlock()` uses it to render one building as a merged block (box-shadow fills the grid gaps, label only on the top-left cell), and `assignBlockIds()` retrofits bids onto presets/old saves by greedily matching w×h chunks. `checkSpots()` also feeds the sidebar stats panel (`renderSpotStats`: active spots + near-miss "differ by one facility" list). The `◆ 視角` button toggles `body.diamond` — the grid rotates 45° via CSS (labels counter-rotated) to mimic the game's isometric view, with `fitGrid()` sizing the rotated square.

- **Placement suggestions**: `updateSuggestions()` marks empty tiles with a dashed blue outline (`.suggest`) where placing the currently selected building would complete a popularity spot — i.e., a 5×5 window already contains every other required item of some spot. Recomputed in `checkSpots()` and on `selectTool()` (which also toasts the completable spot names).
